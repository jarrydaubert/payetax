# Reports

Reports are dated evidence, not evergreen guidance.

Use this area for release verification records and other snapshots where the date, version, command output, or manual confirmation matters. Do not edit old reports just because a command or workflow changed later; add a new report or update the active guide instead.

For evergreen release and production validation guidance, use:

- `docs/guides/OPERATIONS.md`
- `docs/guides/TESTING.md`
