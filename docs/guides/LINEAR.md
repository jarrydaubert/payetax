# Linear Guide

Linear is for tracking work that already exists in the backlog. The backlog remains the source of truth.

## When to Create an Issue

- User-facing bug
- Go-live blocker
- Work item already listed in `docs/BACKLOG.md`

## How to Write Issues

- Use clear, action-oriented titles
- Link to relevant code, docs, or screenshots
- Keep scope tight; split large efforts into smaller issues

## Workflow

- Keep issue state accurate
- Close issues only when the change ships and is covered by a test

## Notes

If work is not in the backlog, add it there first.
